# 生态世界沙盒 · 项目文档

> 上帝视角的种族生态沙盒：程序化世界自运行（繁衍 / 扩张 / 战争 / 兴衰），玩家用天气、地形、疾病等神力干预进程。
> 本文档用于**沉淀已实现内容**，便于后续提新需求时快速建联定位。

---

## 1. 项目简介

| 项 | 说明 |
|---|---|
| 类型 | 上帝视角生态沙盒 / 程序化模拟 |
| 形态 | 单文件 HTML（Canvas + 原生 JS，无依赖，浏览器直接打开） |
| 主程序 | [`生态世界沙盒.html`](生态世界沙盒.html) |
| 核心循环 | `requestAnimationFrame` 驱动；世界自运行 + 玩家实时干预 |
| 设计原型 | 精灵(森林) / 矮人(山地) / 人马(草原) / 鳄鱼(沼泽) 四族 |

### 玩法一句话
程序化生成地图 → 四族小人按 biome 分布并自主游走 → 出生/死亡的人口循环 → 领地扩张 → 接壤战争；玩家随时用 6 种神力改写进程。

## 4. 核心功能模块（10 大模块）

### 4.1 程序化地图
- **噪声**：值噪声 `vnoise` + 5 层 `fbm`（整数哈希 `ihash`）
- **属性**：海拔 `elev` / 湿度 `moist` / 温度 `temp`（温度 = 1 − 纬度×1.45 − 海拔×0.35）
- **地图类型**（海拔衰减规律）：大陆 / 群岛 / 盆地内海 / 超大陆
- **Biome 映射** `biomeOf(e,m,t)`：14 种地貌（见 §6）
- **河流**：高海拔点沿最陡下降流淌入海
- 渲染到**离屏 `bg` canvas**（每帧复用，性能高）

### 4.2 小人（Agent）
- 按各族 **栖息地 biome**（`HAB`）初始放置
- **AI 游走**：`pickTarget` 在 home 附近随机选同 biome 目标 → 直线移动 → 到达休息 → 换目标
- **迷你像素绘制** `drawAgent`：四族差异化剪影 + 行走腿交替 + 身体起伏 + 朝向翻转 + 状态标记（瘟疫红点/饥饿黄点/交战火花）
- 按 y 排序绘制实现**远近遮挡**

### 4.3 人口动态（每 `ECO_INTERVAL=18` 帧）`ecoTick`
- **繁殖**：Logistic `r_eff = r×(1 − n/K)`，新生儿在父母附近同 biome 出生
- **死亡**：年老（age>maxAge）/ 饥饿（人口过载 → hunger 飙升）/ 瘟疫 / 小概率自然
- **承载力 K**：`KCAP[race] = 栖息地 tile 数 × 0.5`
- **总量上限** 650（防卡顿）

### 4.4 瘟疫 `maybePlague`
- 随机偶发（0.5%/eco）选一族感染一批小人，感染期内额外死亡率
- 也可由玩家神力主动施放

### 4.5 领地与扩张
- `computeTerritory`：小人所在格及周围 1 格标为本族领地（后写覆盖 → 接壤区归后者）
- `renderTerritory`：半透明族色叠加层（可开关）
- `expansion`：人口压力 `n/K > 0.8` 时派「开拓者」去 `findFrontier` 找的未占适宜新地 → 领地蔓延

### 4.6 侵略 / 战争（每 10 帧）`combatTick`
- **空间网格**找邻近（<1.1 格）异族小人，35% 概率交战
- **战力** `STR`：矮人 1.5 > 人马 1.2 > 精灵 1.0 > 鳄鱼 0.8；`战力×(0.5~1.5随机)` 定胜负，败者阵亡
- 头顶火花特效；单轮阵亡≥4 弹消息

### 4.7 玩家神力 `applyPower`（鼠标点击/拖动）
| 神力 `power` | 效果 |
|---|---|
| `rain` 降雨 | 湿度↑ → 变森林/绿洲 |
| `raise` 升地 | 海拔↑ → 造山隔断 |
| `water` 挖水 | 海拔↓ → 造河造海隔离 |
| `forest` 造林 | 湿度↑ + 直接变森林 |
| `plague` 瘟疫 | 范围内小人染病 |
| `meteor` 灾祸 | 范围内小人灭杀 + 地表变火山焦土 |
- `forEachTile` 圆形笔刷（半径 `brushR` 1~6）；改地形后 `recalc` 重算 biome + `renderBg`

### 4.8 建筑/聚落系统
- `checkSettlements`：同格 ≥5 同族小人 + 适宜 biome → 生成 1 级聚落；冷却 12 次 `ecoTick`，同族间距 ≥5 格
- 三级聚落：1级小屋（≥5人）→ 2级中房+旗帜（≥9人）→ 3级城堡+塔楼（≥18人）
- 繁殖加成：聚落 4 格内 `rEff × (1 + 0.12 × level)`
- 战损：异族在聚落 1.6 格内交战获胜 → HP -35~-65；HP≤0 时聚落被摧毁
- `meteor` 神力可直接夷平范围内聚落
- `drawSettlement`：像素建筑绘制 + HP<60 红色受损 overlay

### 4.9 实时人口曲线图
- 位置：地图右侧，300px 宽 × 702px 高（与地图同高）
- 每 `ECO_INTERVAL=18` 帧采样一次各族人口，保存最近 `MAX_HIST=300` 个点
- 四族折线（各族 HUD 色），末端带深色底数值标签
- Y 轴动态计算，最小基准 650，向上取整到 50 的倍数

### 4.10 胜负判定 + 事件流面板
- **左侧事件流**：200px 面板，记录最近 15 条事件，带 `分:秒` 时间戳
- **事件分级**：普通(•) / 警告(⚠) / 里程碑(★) / 结局(◆)，四级颜色区分
- **胜负规则**：单族灭绝只记录事件，不结束游戏；**只剩一个种族**时弹出「🏆 世界统一」结局面板
- **结局面板**：存活帧数 / 剩余人口 / 聚落总数 / 四族最终人口；提供「🔄 再来一局」按钮
- **事件埋点**：聚落建立/升级、人口破百、大规模交战、聚落攻陷、瘟疫、灾祸、统一/灭绝

## 5. 技术架构

### 5.1 Agent 数据结构
```js
{
  race,            // 'forest'|'dwarf'|'nomad'|'marsh'
  seed,            // 随机相位（动画/造型）
  x, y,            // tile 浮点坐标
  homeI, homeJ,    // 家园 tile（扩张时迁移）
  tx, ty,          // 当前移动目标
  facing,          // 1 右 / -1 左
  moving, rest,    // 移动状态 / 休息计时
  age, hunger,     // 年龄 / 饥饿值
  plague, fight    // 瘟疫计时 / 交战特效计时
}
```

### 5.2 全局状态
```
W=96, H=54, CELL=13              // 地图格子数 / 每格像素
elev[], moist[], temp[]         // 地块属性（长度 W*H）
biome[], river[]                // biome 字符串 / 河流标记
agents[]                        // 所有小人
terr[]                          // 领地归属（0 无主 / 1-4 族）
KCAP{}                          // 各族承载力
bg, terrCv                      // 离屏画布：地图层 / 领地层
settlements[]                   // 聚落数组（含 level/hp）
popHistory[]                    // 人口历史（用于右侧图表）
eventLog[]                      // 事件流历史（左侧面板）
prevPop{}                       // 上轮人口（用于灭绝检测）
gameOver                        // 胜负已分标记
power, brushR, hoverI/J, painting // 神力交互状态
frame                           // 全局帧计数
```

### 5.3 主循环 `tick()`（rAF）
```
if(!paused){
  frame++
  for agent: fight--；rest 或 朝目标移动（facing 跟随）
  frame % 10  === 0 → combatTick()    // 战争
  frame % 18  === 0 → ecoTick()       // 人口/扩张/领地
}
draw()
```

### 5.4 渲染分层 `draw()`
```
drawImage(bg)            // 地图（离屏缓存）
drawImage(terrCv)        // 领地半透明层（可开关）
for s in settlements: drawSettlement(ctx,s)   // 聚落建筑
agents.sort(y).forEach(drawAgent)   // 小人（含动画/状态）
笔刷光标圆圈              // 神力 hover 预览
drawChart()              // 右侧人口曲线图（仅在 ecoTick 后重绘）
```

## 6. 关键参数速查表

### 种族参数
| 种族 | key | 栖息地 HAB | 繁殖 r | 寿命 maxAge | 战力 STR | 迷你色 |
|---|---|---|---|---|---|---|
| 精灵(森林) | forest | FOREST, PLAIN | 0.030 | 2600 | 1.0 | #5bbf5a |
| 矮人 | dwarf | MOUNT, HILL | 0.016 | 3200 | **1.5** | #caa36a |
| 人马(游牧) | nomad | GRASS, PLAIN | 0.022 | 2400 | 1.2 | #cf8a4a |
| 鳄鱼(沼泽) | marsh | SWAMP, BEACH | **0.040** | 2000 | 0.8 | #3fa07a |

### Biome 映射 `biomeOf(e,m,t)`（e=海拔 m=湿度 t=温度）
```
e<0.34 → DEEPSEA      e<0.40 → SEA        e<0.435 → BEACH
e>0.90 → SNOW/ROCK(t<0.32)   e>0.78 → MOUNT     e>0.70 → HILL
t<0.26 → SNOW
e<0.50 且 m>0.66 → SWAMP
m<0.24 → DESERT   m<0.46 → GRASS   m<0.68 → PLAIN   否则 → FOREST
```

### 地图类型（海拔衰减，d=离中心归一距离）
| 类型 | 公式 |
|---|---|
| continent 大陆 | e − d×0.55 + 0.12 |
| islands 群岛 | e×1.15 − 0.22 |
| basin 盆地 | e + (d−0.5)×0.55 |
| pangaea 超大陆 | e − d×0.30 + 0.18 |

### 关键时序 / 阈值
| 参数 | 值 | 含义 |
|---|---|---|
| ECO_INTERVAL | 18 帧 | 人口动态周期 |
| 战斗检测 | 每 10 帧 | combatTick |
| 战斗范围 / 概率 | <1.1 格 / 35% | 接壤交战 |
| 承载力系数 | ×0.5 | KCAP = 栖息地 tile × 0.5 |
| 扩张阈值 | n/K > 0.8 | 触发开拓者 |
| 总人口上限 | 650 | 防卡顿 |
| 笔刷半径 | 1~6 | brushR |
| 聚落生成人口 | ≥5 同族/格 | 触发新聚落 |
| 聚落生成冷却 | 12 次 ecoTick | 约 216 帧 ≈ 3.6s |
| 聚落最小间距 | 5 格 | 同族聚落之间 |
| 聚落繁殖加成 | +12%/lv | 1级+12%，3级+36% |
| 聚落战损 | 35~65 HP/次 | 异族交战获胜 |
| 图表历史容量 | 300 点 | 约 90 秒 @60fps |
| 事件流容量 | 15 条 | 左侧面板滚动 |
| 统一胜利条件 | 只剩 1 个种族 | 其他三族全部灭绝 |

### 接入新种族的完整清单（避免遗漏）
新增一族需同步更新：`HAB`、`RACE_NAME`、`RACE_PARAMS`、`STR`、`RACE_IDX`、`TERR_COL`、`RACE_COL`、`drawAgent` 的 switch 分支、HUD pop 统计对象。

### 技术约束备忘
- 单文件、无依赖、原生 Canvas；保持这一形态便于直接打开调试
- 性能敏感处：`combatTick`（空间网格已优化）、`renderBg`（仅地形变更时重渲染）
- 坐标体系：小人用 **tile 浮点坐标**，绘制时 ×`CELL`；鼠标用 `getBoundingClientRect` 换算（适配缩放）

### 13.2 战斗升级系统（新增大模块）

#### 13.2.1 设计目标
解决原战斗系统三大问题：
1. **一击必杀无翻盘** → 引入等级战力乘数，老兵可逆袭
2. **无个体成长** → 击杀积累经验，晋升变强
3. **击杀无反馈** → 连斩奖励+升级事件+王牌单位追踪

#### 13.2.2 四阶晋升体系

| 等级(rank) | 精灵 | 矮人 | 人马 | 海妖 | 人类 | 战力乘数 |
|:---:|:---|:---|:---|:---|:---|:---:|
| **0** | 嫩芽 | 学徒 | 幼驹 | 幼体 | 民兵 | ×1.0 |
| **1** | 游侠 | 锻铁者 | 风行者 | 深潜者 | 骑士 | **×1.4** |
| **2** | 古树守卫 | 战争领主 | 战蹄酋长 | 潮汐领主 | 军团长 | **×1.9** |
| **3** | 翡翠君王 | 山王 | 草原霸主 | 潮汐之母 | 至高统帅 | **×2.6** |

> **称号命名原则**：各族独立，霸气但不越界（禁用"神"字），4字以内便于HUD显示。

#### 13.2.3 战力公式重构

```js
// 旧
const sa = STR[a.race] * (0.5 + Math.random());
// 新
const sa = STR[a.race] * RANK_MULT[a.rank] * (0.5 + Math.random()) * auraBonus;
```

**质变示例**：传说精灵翡翠君王（1.0×2.6=2.6）可正面碾压普通矮人（1.5×1.0=1.5）。

#### 13.2.4 击杀经验矩阵

基础XP = 矩阵值 × `STR[目标种族]`（种族价值系数）

| 击杀者＼目标 | 0阶 | 1阶 | 2阶 | 3阶 |
|:---:|:---:|:---:|:---:|:---:|
| **0阶** | 1 | 2 | 5 | 10 |
| **1阶** | 1 | 1 | 3 | 6 |
| **2阶** | 1 | 1 | 1 | 3 |
| **3阶** | 1 | 1 | 1 | 1 |

> 越级击杀重奖，虐菜无收益。击杀传说获得10XP（但传说极难死）。

#### 13.2.5 连斩奖励

30帧内连续击杀：
- 2杀：XP ×1.3
- 3杀：XP ×1.6
- 4杀+：XP ×2.0

事件示例：`🔥 矮人山王连斩4人！`

#### 13.2.6 高等级限制机制（核心设计）

| 等级 | 限制类型 | 规则 |
|:---:|:---|:---|
| **3阶 传说** | **硬性唯一制** | 每族最多1个；需满足：① 本族无传说 ② 击杀过敌方英雄 ③ 本族有≥2级聚落 |
| **2阶 英雄** | **人口比例软限制** | ≤ 该族总人口15%；超限后勇士XP获取-50% |
| **1阶 勇士** | **人口比例软限制** | ≤ 该族总人口50%；超限后学徒XP获取-30% |
| **0阶 学徒** | 不限 | 新生儿默认学徒 |

> 软限制通过"瓶颈"实现：不硬卡数量，但超额后升级效率暴跌，自然淘汰后自动回落。

#### 13.2.7 配套机制
