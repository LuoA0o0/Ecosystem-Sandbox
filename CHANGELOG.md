### 1. 建筑/聚落系统

**目标**：让小人有"家"的概念，从生态模拟迈向文明模拟。

| 项目 | 实现 |
|---|---|
| 数据结构 | `settlements[]`，字段：`{i,j,race,age,pop,level,hp}` |
| 生成条件 | 同格 ≥5 同族小人 + 适宜 biome + 非河流 |
| 生成冷却 | 每 12 次 `ecoTick`（约 216 帧 ≈ 3.6 秒）检查一次 |
| 密度限制 | 同族聚落间距 ≥5 格 |
| 等级系统 | 1级（≥5人）→ 2级（≥9人）→ 3级（≥18人） |
| 繁殖加成 | 聚落 4 格内：`rEff × (1 + 0.12 × level)` |
| 战损 | 异族在聚落 1.6 格内交战并获胜 → HP -35~-65 |
| 神力摧毁 | `meteor` 范围内聚落直接夷平 |
| HUD | 各族显示 `🏠N` 聚落数量 |

**新增函数**：
- `checkSettlements()` — 生成新聚落 + 动态升级
- `cleanupSettlements()` — 清理 HP≤0 的聚落
- `drawSettlement(ctx, s)` — 三级像素建筑绘制 + 受损红 overlay

**关键改动位置**：
- `initAgents()` — 重置 `settlements=[]`
- `ecoTick()` — 繁殖时接入 `settleBonus`，末尾调用 `checkSettlements()`
- `combatTick()` — 战斗后扣聚落 HP
- `applyPower(meteor)` — 摧毁范围内聚落
- `draw()` — 在 agents 之前渲染聚落

### 2. 实时人口曲线图

**目标**：让玩家直观观察四族兴衰、战争节点、神力干预效果。

| 项目 | 实现 |
|---|---|
| 位置 | **地图右侧**，竖向排列 |
| 尺寸 | 300px 宽 × 702px 高（与地图同高） |
| 采样频率 | 每 `ECO_INTERVAL=18` 帧（即每次 `ecoTick`） |
| 历史容量 | `MAX_HIST=300` 个点，满后滚动 |
| Y轴 | 动态计算，最小基准 650，向上取整到 50 的倍数 |
| 视觉 | 6 条网格线 + 四族折线 + 末端深色底数值标签 |

**关键改动位置**：
- HTML — `<canvas id="chart">` 插入 flex 布局
- `drawChart()` — 新增绘制函数
- `ecoTick()` — 末尾 `popHistory.push()` + `drawChart()`
- `initAgents()` — 重置 `popHistory=[]`

### 3. 胜负判定 + 左侧事件流（方案A）

**目标**：给无尽沙盒一个终点，同时让玩家不错过任何史诗时刻。

#### 3.1 左侧事件流面板

| 项目 | 实现 |
|---|---|
| 位置 | **地图左侧**，200px 宽，与地图同高 |
| 容量 | `MAX_EVENTS=15` 条，自动滚动 |
| 时间戳 | `分:秒` 格式 |

**事件分级**：

| 类型 | 图标 | 颜色 | 触发场景 |
|---|---|---|---|
| 普通 | • | `#9aa0b0` | 建立新聚落 |
| 警告 | ⚠ | `#e0b020` | 大规模交战、瘟疫、聚落攻陷、灾祸 |
| 里程碑 | ★ | `#4fc3f7` | 聚落升级、人口破百 |
| 结局 | ◆ | `#ff2442` | 种族灭绝（仅记录）、世界统一 |

**新增函数**：
- `pushEvent(type, text)` — 压入事件并刷新面板
- `renderEventLog()` — 渲染事件列表到 `#eventList`

#### 3.2 胜负判定

| 规则 | 说明 |
|---|---|
| 单族灭绝 | **不结束游戏**，仅记录红色事件到左侧面板 |
| 只剩一个种族 | 弹出中央 **「🏆 世界统一」** 结局面板，自动暂停 |
| 领地占比 > 75% | **已移除**，改为纯灭绝制胜利 |

**结局面板内容**：
- 标题：🏆 世界统一（获胜种族颜色）
- 描述：XX 淘汰了所有对手，成为世界的唯一主宰
- 统计：存活帧数 / 剩余人口 / 聚落总数 / 四族最终人口
- 按钮：🔄 再来一局 → 自动生成新 seed 重建世界

**新增函数**：
- `checkVictory()` — 每轮 `ecoTick` 检查灭绝/统一/里程碑
- `showGameOver(race, pop)` — 弹出中央模态框

**事件埋点汇总**：

| 触发位置 | 事件内容 | 级别 |
|---|---|---|
| `checkSettlements` | XX 建立了新聚落 | 普通 |
| `checkSettlements` | XX 聚落升至 2/3 级 | 里程碑 |
| `ecoTick` | XX 人口突破 100 | 里程碑 |
| `combatTick` | 领地交战，N 人阵亡 | 警告 |
| `combatTick` | XX 的聚落被攻陷 | 警告 |
| `maybePlague` | XX 爆发瘟疫，感染 N 人 | 警告 |
| `applyPower(meteor)` | 天降灾祸，N 人罹难 | 警告 |
| `applyPower(meteor)` | XX 的聚落被灾祸夷平 | 警告 |
| `checkVictory` | XX 已灭绝 | 结局（仅记录） |
| `checkVictory` | XX 统一了世界！ | 结局（弹窗） |

### 4. Bug 修复

#### 再来一局按钮未生效
- **原因**：`#modalRestart` 按钮的 HTML 放在 `</script>` 之后，JS 绑定事件时元素尚未存在于 DOM
- **修复**：将模态弹窗 HTML 移至 `<script>` 标签之前
